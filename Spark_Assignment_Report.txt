Spark Architecture

Apache Spark follows a master-worker architecture consisting of: -
Driver Program: Creates the SparkSession, builds the DAG, and schedules
tasks. - Cluster Manager: Allocates resources to executors. - Executors:
Execute tasks, process data, and return results.

Performance Concepts

-   Lazy Evaluation delays execution until an action is called.
-   DAG optimizes execution by combining transformations.
-   GroupBy causes Shuffle.
-   Predicate Pushdown improves query performance.
-   Parquet generally performs better than CSV due to columnar storage
    and compression.

Results

The assignment successfully demonstrated Spark DataFrame operations,
schema handling, transformations, actions, filtering, aggregation, and
performance optimization. Writing CSV and Parquet files could not be
demonstrated because Hadoop (HADOOP_HOME/winutils.exe) was not
configured on the Windows environment.

Best Practices

-   Read only required columns.
-   Filter data as early as possible.
-   Handle null values before analysis.
-   Prefer show() over collect() for large datasets.
-   Use Parquet for analytical workloads.
-   Minimize unnecessary shuffle operations.

Conclusion

This assignment provided practical experience with Apache Spark and
PySpark. It demonstrated how Spark processes data efficiently using lazy
evaluation and optimized execution plans while applying common DataFrame
operations and performance best practices.
